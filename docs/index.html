A practical app layer for a Linux VPS.

Vessel keeps the runtime small: a Go binary, SQLite metadata, Docker Compose files on disk, and a public catalog served by GitHub Pages.

Single-source templates, cloud-served updates.

Contributors add one YAML file under internal/registry/templates. Release binaries embed that same catalog, while GitHub Pages publishes a compact JSON index containing the YAML payload, metadata, and hashes.

1

A contributor opens a PR with one template YAML file.

2

Pages builds templates/index.json from that source folder.

3

Running Vessel servers fetch one small catalog and merge it over the embedded fallback.
